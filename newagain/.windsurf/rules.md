| 省 | 职责 | 输出 |

# 项目上下文

## 项目目标

优化 YOLO11n 模型，引入 ARConv（自适应感受野卷积），使其在嵌入式设备上实现高精度、低延迟的目标检测。

## 当前阶段：训练与实验

### 对比实验阵容（7个n级别轻量模型）

| # | 模型 | 参数量 | 状态 |
|---|------|--------|------|
| 1 | YOLOv5n | ~1.9M | 待建脚本 |
| 2 | YOLOv8n | ~3.2M | 已有结果 |
| 3 | YOLOv10n | ~2.3M | 已有结果 |
| 4 | YOLO11n（基线） | ~2.6M | 已有结果 |
| 5 | YOLO12n | ~2.6M | 待重跑 |
| 6 | YOLOv13n | ~2.3M | 需安装第三方 |
| 7 | YOLO26n | ~2.6M | 需添加yaml |

### 统一超参

```
epochs=150, batch=16, imgsz=640, cache=True
close_mosaic=10, workers=8, optimizer=SGD
amp=True, device='0'
```

### 消融实验

- ARConv 各组件消融（位置、数量、组合）
- 结果目录：runs/ablation/

### 叙事目标

对比实验结论自然导向"选择YOLO11n作为部署基线是合理的"——同量级公平比较 + 嵌入式部署视角。

## 关键路径

```
ultralytics-main/
├── cfg/models/          # 各版本模型yaml
├── runs/                # 训练结果
├── ultralytics/nn/      # 核心模块（含ARConv实现）
└── train_*.py           # 各模型训练脚本
```

## 科研纪律

1. **可复现**：每次实验记录完整超参、种子、数据版本
2. **公平比较**：统一超参、统一数据集、统一评估协议
3. **不跳步**：先基线后改进，先消融后组合
4. **留痕**：实验结果即时记录到 `旧实验结果与问题记录.md`
5. **最小变更**：改一处验证一处，不批量盲改
6. **异常即止**：训练loss异常、精度骤降，先排查再继续
