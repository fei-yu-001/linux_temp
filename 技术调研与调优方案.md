# 天气图像分类技术调研与调优方案

> 项目：睿抗天气图像分类（4 类：cloudy/rainy/snowy/sunny）
> 评测指标：F1 macro
> 当前 baseline：EfficientNet-B0/B2 预训练微调 ≈ 0.94
> 目标：冲到 0.96+
> 约束：本地 CPU 训练（速度无所谓），平台支持自己上传数据集

---

## 一、数据集现状分析

### 1.1 类别分布

| 类别 | 样本数 | 占比 | 平均图片大小 |
|---|---|---|---|
| cloudy | 2184 | 43.7% | 13.8 KB |
| sunny | 1966 | 39.3% | 15.0 KB |
| rainy | 446 | 8.9% | 15.2 KB |
| snowy | 403 | 8.1% | 15.5 KB |
| **合计** | **4999** | 100% | — |

### 1.2 关键问题

- **不平衡比 ≈ 5.4:1**（cloudy:snowy），属**中等不平衡**
- rainy/snowy 是少样本类，也是 F1 macro 的主要拖累项
- 数据量仅 5000 张，属**小数据集**
- 图片平均大小 13-15KB，分辨率可能偏低

### 1.3 数据集优化方向

| 方法 | 说明 | 预期收益 |
|---|---|---|
| **标签清洗** | cleanlab 自动检测标注错误（5000 张通常 2-5% 标错） | +0.5~1% |
| **Diffusion 合成少数类** | SD + ControlNet 生成 rainy/snowy 各 200-500 张 | +0.3~1%（需 A/B 验证） |
| **半监督学习** | FixMatch/FreeMatch 利用额外无标签天气图 | +0.5~1.5% |
| **重划分数据集** | 如果原始划分不合理，重新分层划分 | 验证集更稳定 |

## 二、技术选型调研（2024-2026 开源 SOTA）

### 2.1 Backbone 选型

| 模型 | 来源 | 参数量 | ImageNet top-1 | 适合本任务 | 备注 |
|---|---|---|---|---|---|
| EfficientNet-B0/B2（baseline） | torchvision | 5-9M | 77-80% | ⭐⭐⭐ | 当前已用 |
| **ConvNeXt V2-Base** | timm | 89M | 83.9% | ⭐⭐⭐⭐⭐ | 纯 CNN SOTA，迁移友好 |
| **DINOv2 ViT-L/14** | timm/torch.hub | 300M | 线性探针顶级 | ⭐⭐⭐⭐⭐ | 小数据杀手锏，冻结+linear head |
| **DINOv3**（2025-08） | Meta | 7B | 顶级 | ⭐⭐⭐⭐ | 最新自监督，资源消耗大 |
| EfficientNetV2-S | torchvision | 21M | 83.9% | ⭐⭐⭐ | baseline 同源 |
| MaxViT-Base | timm | 31M | 83.6% | ⭐⭐⭐⭐ | 多轴注意力，与 CNN 互补 |
| DeiT III | timm | 22M | 83%+ | ⭐⭐⭐ | ViT 改进训练配方 |
| Swin-Tiny | torchvision | 28M | 81.3% | ⭐⭐ | 小数据易过拟合 |

**核心结论**：
- **DINOv2 ViT-L/14**：冻结 backbone + linear head，5000 张小数据上往往优于暴力微调
- **ConvNeXt V2**：纯卷积 SOTA，与 EfficientNet 同系但更强
- **MaxViT**：多轴注意力，与 CNN 互补，适合集成
- 需要安装 **timm** 库（当前未安装）

### 2.2 训练技巧（Bag of Tricks）

| 技巧 | 来源 | 效果 | 实现难度 | CPU 成本 |
|---|---|---|---|---|
| Label Smoothing(0.1) | 经典 | +0.2~0.5% | 极低 | 0 |
| **EMA**（指数移动平均） | timm 内置 | +0.2~0.5% | 极低 | ~0 |
| **SWA**（随机权重平均） | PyTorch 原生 | +0.3~0.8% | 低 | ~0 |
| Cosine LR + Warmup | 标准 | 稳定收敛 | 低 | 0 |
| Mixup(α=0.4) | timm 内置 | +0.2~0.8% | 低 | 0 |
| CutMix(α=1.0) | timm 内置 | +0.2~0.8% | 低 | 0 |
| RandAugment(n=2,m=9) | timm 内置 | +0.3~1% | 低 | 0 |
| RandomErasing | timm 内置 | +0.1~0.3% | 低 | 0 |
| **SAM 优化器** | [GitHub](https://github.com/davda54/sam) | +0.3~0.8% | 中 | 2× 训练 |
| 差分学习率 | Kaggle 套路 | +0.2~0.5% | 低 | 0 |

### 2.3 类别不平衡处理

> 不平衡度 5.4:1 属中等，不需要重型长尾方法。

| 方法 | 来源 | 效果 | 实现难度 | 备注 |
|---|---|---|---|---|
| **WeightedRandomSampler** | PyTorch 原生 | +0.2~0.5% | 极低 | 按 1/√n 加权 |
| **Class-Balanced Loss** | [CVPR 2019](https://github.com/yinwenpeng/ClassBalancedLoss) | +0.3~0.8% | 低 | 有效样本数重加权 |
| **Logit Adjustment** | [ICLR 2021](https://github.com/google-research/google-research/tree/master/logit_adjustment) | +0.3~1% | 极低 | **专治 F1 macro** |
| Focal Loss(γ=1.5) | 经典 | +0.2~0.5% | 低 | 难样本加权 |
| LDAM-DRW | [NeurIPS 2019](https://github.com/kaidic/LDAM-DRW) | +0.3~0.8% | 中 | margin + 延迟重加权 |
| **阈值搜索** | 实战经验 | +0.5~1% | 极低 | 每类独立 threshold 调优 |

**推荐组合**：WeightedRandomSampler + Class-Balanced Loss + Logit Adjustment + 阈值搜索

### 2.4 测试时增强（TTA）

使用 [ttach](https://github.com/qubvel/ttach) 库：

| 策略 | 效果 | 耗时 |
|---|---|---|
| HorizontalFlip | +0.2~0.4% | 2× |
| MultiScale([0.9,1.0,1.1]) | +0.3~0.6% | 3× |
| **HFlip + MultiScale** | +0.5~0.8% | 6× |
| + FiveCrops | +0.3~0.5% | 30× |

**推荐**：HFlip + MultiScale（性价比最高，CPU 完全可接受）

### 2.5 集成学习

| 方法 | 效果 | 训练开销 | 推理开销 | 备注 |
|---|---|---|---|---|
| **Model Soup** | +0.5~1.5% | n× | 1× | **权重平均，推理不增加** |
| Logits/Softmax 集成 | +0.5~1.5% | n× | n× | 互补模型平均概率 |
| Seed Averaging | +0.3~0.8% | n× | n× | 同模型多 seed |
| Stacking | +0.3~0.8% | 需 OOF | 中 | 4 类小数据易过拟合 |

**推荐**：
- **Model Soup**（同架构多 seed/超参 → 权重平均，推理零开销）
- **多模型 logits 集成**（DINOv2 + ConvNeXt V2 + MaxViT，3 个互补架构）

## 五、关键提醒

1. **不平衡度 5.4:1 属中等**，先做 Logit Adjustment + WeightedSampler，不需要 PaCo/BCL 重型方法
2. **F1 macro 与阈值强相关**，推理后每类 threshold 做 grid search 常常免费 +0.5~1%
3. **颜色敏感任务**：避免过强 color jitter（可能把 sunny 变 cloudy），几何增强更安全
4. **DINOv2 linear probe 是隐藏杀器**：5000 张小数据上先抽特征 + linear head，比暴力微调更稳
5. **Model Soup 是免费午餐**：同架构多 seed 微调后权重平均，推理零开销，几乎必赚
6. **高分辨率迁移**：天气细节（云纹理、雨丝、雪花）对分辨率敏感，384 比 224 通常涨 0.5~1%
7. **当前未安装 timm**，需要先 `pip install timm`
